# 📝 智能工位系统：架构升级与接口交接文档

## 一、 方案重大调整说明（数字孪生/虚拟仿真）

为了应对答辩现场网络环境复杂、无物理红外发射/接收元件、以及不方便组装大功率强电电器（台灯、空调）**的客观限制，本系统执行机构全面升级为**“数字孪生（Digital Twin）虚拟仿真”方案：

1. **解耦物理依赖**：取消原本的物理红外收发和继电器硬件，台灯与空调的运行状态完全由底层 Arduino 的状态机在**软件层面**计算。
2. **Web 虚拟响应**：当状态触发时，主控板通过无线网络向网页端推送控制报文，由网页前端（HTML/JS）以**全动态 UI、动画特效**的形式无延迟仿真呈现设备的开关状态。
3. **AP 热点无网演示**：ESP32-C6 将开启 **AP 模式（自建 Wi-Fi 热点）**，答辩现场电脑直接连接 ESP32 的 Wi-Fi 访问本地网页，**100% 免疫弱网或无网环境**。

## 二、 硬件连接微调（极简布线）

由于去除了红外、继电器等复杂的输出外设，目前本地感知层硬件已实现最大化精简，彻底避免了面包板排线松动、没有 $10\text{k}\Omega$ 物理电阻的困扰：

* **超声波传感器**：`Trig` $\rightarrow$ `D2`，`Echo` $\rightarrow$ `D3`
* **DS18B20 温度传感器**：数据线 $\rightarrow$ `D4`
* **光敏电阻**：直插 `A0` 与 `GND`（已在 Arduino 内部配置 `INPUT_PULLUP` 软件上拉，$10\text{k}\Omega$ 内部电阻已生效）。
* **主控间通信**：Arduino Uno 与 ESP32-C6 之间通过 **USB 数据线/串行总线** 物理连接。

## 三、 Arduino 核心代码逻辑变更

我这边负责的底层感知码已完成全面重构，主要有以下三点改动：

1. **连根拔除红外库**：彻底删除了 `IRremote` 库及引脚轮询，释放了硬件定时器 `Timer2`，系统运行更流畅，彻底避免了引脚悬空产生的静电杂讯干扰。
2. **环境光阈值适配**：根据实测，当前光敏电路在**有灯光时 L 约为 380**，**手捂遮光时 L 约为 650**。因此，代码中环境昏暗阈值已精准微调为 **`500`**。
3. **新增空调状态机变量**：在软件内部维护一个 `isAcOn` 变量（开机默认值为 `true`）。
* 当超声波判定离座超时满 5 分钟（300000ms）时，`isAcOn` 自动切为 `false`。
* 当用户重新回到座位时，`isAcOn` 自动恢复为 `true`。

## 四、 核心数据接口协议（API）数据交接

这是我们模块间协同的核心。Arduino 每隔 **`200ms`** 会通过串口（Serial）以 **`115200`** 波特率向上游网关（ESP32-C6）高频吐出一行标准的文本报文。

### 📡 报文格式示例：

`D:100,L:380,T:25.0,S:1,L_ST:0,AC_ST:1`

### 📊 字段解析对照表（供后续同学解析字符串）：

| 字段名 | 含义 | 数据类型 | 取值范围与逻辑说明 | 前端仿真响应建议 |
| --- | --- | --- | --- | --- |
| **`D:`** | 用户距离 | 整型 (int) | 单位：cm（经过 5 组滑动窗口中值平均滤波） | 可在网页以仪表盘/数字显示 |
| **`L:`** | 环境光强 | 整型 (int) | 数值越小环境越暗 | 可在网页以进度条/数字显示 |
| **`T:`** | 当前温度 | 浮点 (float) | 单位：℃（来自 DS18B20） | 网页数字温度计显示 |
| **`S:`** | 系统状态机 | 整型 (int) | **0**：离座节能状态<br>

<br>**1**：正常工作状态<br>

<br>**2**：驼背/离得太近警告状态 | **S=2时**：网页弹出红色全屏高危警告<br>

| **`L_ST:`** | **虚拟台灯状态** | 布尔/整型 | **1**：开启状态<br>

<br>**0**：关闭状态<br>

<br>*(当 S≠0 且 L>500 时自动触发1)* | **L_ST=1时**：网页台灯图标切换为金黄色发光特效<br>

<br>**L_ST=0时**：网页台灯图标切换为灰色 |
| **`AC_ST:`** | **虚拟空调状态** | 布尔/整型 | **1**：开启状态<br>

<br>**0**：关闭状态<br>

<br>*(当 S=0 且维持满 5 分钟时自动触发0)* | **AC_ST=1时**：网页空调图标播放扇叶旋转/冷气动画<br>

<br>**AC_ST=0时**：网页空调扇叶动画停止 |

   - Version: v2.3.7 or higher
   - Purpose: Implements the 1-Wire protocol required to communicate with the DS18B20 temperature sensor.

3. DallasTemperature (by Miles Burton)
   - Version: v3.9.0 or higher
   - Purpose: Provides high-level asynchronous temperature readout functions for the DS18B20 sensor.

[Wiring Reference]
- HC-SR04: Trig -> D2, Echo -> D3
- DS18B20: Data -> D4 (Requires no external pull-up resistor if internal state machine is stable)
- LDR (Photoresistor): PIN 1 -> A0, PIN 2 -> GND (Configured with internal INPUT_PULLUP)
- LCD1602 I2C: SDA -> A4, SCL -> A5, VCC -> 5V, GND -> GND

[Hardware Optimization Note]
- Infrared transmission/reception libraries (e.g., IRremote) are intentionally DEPRECATED and REMOVED in this version to free up Hardware Timer2 and prevent floating-pin noise interference, successfully transitioning the system into a Digital Twin / Virtual Simulation architecture.
